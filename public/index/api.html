API

This API concerns the functions implemented or reimplemented by our team for the visualisation of GO data in a treemap using the library FATuM.
Our code is based on the code written by David Auber and found here : http://www.labri.fr/perso/aperrot/fatum/treedemo/index.html.

If you need the FATuM library API please visit FATuM website.

Function Description Parameters
tmap(elem, depth, x0, y0, y1, s, m) creates a labeled treemap using the data structure passed as elem x0, y0, y1, y1: the four FATuM coordinates for the position of the treemap
s: determines if the nesting of the inner rectangles will be vertical or horizontal
m: the eventual visual element clicked by the user
findChosenTerm(elem, m) searches in the data the term corresponding to the visual
element clicked by the user and adds it as the visual element's attribute.		 elem: the data structure
m: the visual element clicked by the user
createLabel(elem, m, y0, y1) creates a label of the data object elem using its attribute name m: the chosen visual element by the user
y0, y1: coordinates of the visual element representing
elem.
clearLabels() deletes the previous labels when the user has interacted with the treemap.
That way new labels can be placed in different ways.
colorRelatives(elem, m) colors the data object given as elem if it has a parent-child relation with the
visual element chosen by the user m		 elem: the data structure

m: the chosen visual element by the user
updateMarks(m) calls clearLabels() and tmap() to reconstruct the treemap according to
the user's click position.
treeInteractor() determines the actions taken when the user has clicked on the treemap
initVis() initializes the visualisation calling different functions
